# Drop pgx v4 Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove built-in `github.com/jackc/pgx/v4` support, make pgx v5 the only built-in pgx integration, and keep this as a pre-`v1` `v0.2.0` breaking change without changing the module path.

**Architecture:** Keep the public `Database` interface and generic `New` constructor as the custom integration boundary. Replace the pgx compatibility layer with pgx v5-only constructors and a small internal implementation that handles both `*pgx.Conn` and acquired `*pgxpool.Conn` without the old v4/v5 adapter types. Update examples, tests, dependencies, and README to match the new public API.

**Tech Stack:** Go 1.25, `github.com/jackc/pgx/v5`, standard Go tests, existing PostgreSQL integration tests.

---

## File Structure

- Modify `mig_test.go`: remove pgx v4 examples, rename pgx v5 examples to `ExampleFromPgx` and `ExampleFromPgxPool`, and call the new constructors.
- Modify `pgx_internal_test.go`: remove pgx v4 integration coverage and old private-interface assertions; keep pgx v5 internal behavior tests.
- Modify `mig.go`: remove pgx v4 imports and constructors; rename pgx v5 constructors to `FromPgx` and `FromPgxPool`.
- Modify `pgx.go`: remove pgx v4 imports, adapters, and no-rows checks; collapse the private compatibility interfaces where pgx v5 direct types are enough.
- Modify `README.md`: list only pgx v5 as built-in support and document the `v0.2.0` breaking change.
- Modify `docker-compose.yml`: update the integration-test PostgreSQL image to `postgres:18-alpine`.
- Modify `Makefile`: add an 80% coverage gate to the existing `test` target.
- Modify `.github/workflows/pipeline.yaml`: add a blocking `actionlint` job and make the reusable Go check job depend on it.
- Modify `go.mod` and `go.sum`: refresh dependencies with `make update`, then remove `github.com/jackc/pgx/v4` and obsolete indirect dependencies using `go mod tidy`.
- Leave `AGENTS.md` intact unless explicitly asked to commit or edit it; it is user-requested repo policy and currently uncommitted.
